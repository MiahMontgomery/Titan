**Project Name: Titan**

**Titan is** a locally-run AI project management system that takes user-inputted project prompts and autonomously creates, manages, and completes those projects using dedicated AI agents. Titan acts as the central nervous system overseeing all projects and reporting their statuses to the user via Telegram at consistent intervals. It ensures every new project gets its own self-contained agents and interface space, minimizing bottlenecks or overload.

**Titan is NOT** a web app, public-facing SaaS product, or hosted service. It is not designed to use any login, Replit-hosted database, Replit auth, or any built-in deployment systems. It is meant to be downloaded as a ZIP file from Replit, then hosted and executed inside a Google Cloud VM or similar environment where it can connect to external services like Firebase, OpenAI, Telegram, and vector databases.

**Do NOT use:**
- Replit Auth or Login
- Replit DB
- Placeholder APIs or mock integrations
- Hosting features
- App wrappers or publishing tools

---

**Titan WILL:**
- Accept project prompts through a simple text box and auto-create project dashboards
- Automatically spawn 3 dedicated AI agents per project to begin planning, milestone generation, and execution
- Display project activity visually and clearly (active/inactive) on the dashboard
- Relay progress updates to the user every 20 minutes via Telegram (setup stubs)
- Enable future connection of OpenAI GPT-4 Turbo, LangChain, Firebase, and more

**After Titan is complete on Replit:**
- It will be downloaded as a ZIP file
- Unzipped into a Google Cloud VM for 24/7 autonomous execution
- Connected to live APIs, Firebase, Telegram Bot, and local storage for rollback, logging, and memory

**UI Layout Overview**

**Main Dashboard:**
- Displays all current projects in a list/grid format
- Each project tile shows:
  - Project name
  - Activity status (green check = active, black X = inactive)
  - Progress bar (calculated from milestones completed vs. total milestones)
- Top right corner: "Add Project" button to input a name and a text prompt
- Projects are sorted in order of creation or last updated

**Colors & Style:**
- Background: Dark grey (#1f1f1f)
- Text: Light grey (#d3d3d3)
- Accent/Active elements: Bright teal (#01F9C6)
- Active checkmark: Bright teal
- Inactive X: Black
- Logo: Multidimensional 4-point star (resembles a ninja star) in teal and dark blue shades

---

**Project View - Tabs:**

**Tab 1: Progress**
- Main purpose: visualize the current state of a project's development
- Data pulled from: performance logs, milestone tracking, and goal completions generated by the AI agents (once connected post-download)
- Structure:
  - Each project contains a list of "Features" it's working toward
  - Each feature has a dropdown of "Milestones"
  - Each milestone contains "Goals" (subtasks)
  - Estimated hours-to-completion is shown in light gray next to each milestone
- Entire structure is auto-generated and managed by the project agents
- UI will provide dynamic rendering for collapsible lists

**Tab 2: Performance**
- Merged input/output tab
- Top Half:
  - Displays live frontend/visual rendering of the project (even if just a null screen or webpage preview)
- Bottom Half:
  - Real-time display of code being generated, debugged, or logged
  - Code is shown line-by-line (as if using Replit's agent)
  - Logs of debugging attempts, rollbacks, and status updates
  - Always-present text bar at the bottom for agent-to-user and user-to-agent messages or instructions
- This tab functions as the core working space of the project
- Replit should ensure this area is built to run and update continuously once real agents are connected

**Tab 3: Sales**
- Placeholder screen for future PayPal and marketing integration
- Top Half:
  - Platform-by-platform marketing breakdown
  - Engagement and outreach stats (stubbed)
- Bottom Half:
  - Revenue and sales projections
  - Weekly/monthly earnings
- No active data yet; UI elements only

**Post-Zip File Download Requirements**
- Code must be modular and readable
- Must be structured to allow easy connection to:
  - Firebase (for progress, logging, rollback)
  - Telegram Bot (Titan status and alerts every 20 minutes)
  - GPT-4 Turbo via OpenAI API (main reasoning and generation)
  - LangChain (task management and flow)
  - Vector DB (long-term memory and goal reuse)
- Do not bake in any logic that hardcodes local environments or paths
- Leave setup functions like `connectToFirebase()`, `connectToOpenAI()` as stubbed functions

---

**Simplification Decisions:**
- Only 3 tabs per project: Progress, Performance, Sales
- Progress is hierarchical and tied to milestone goals
- Performance merges input/output into a single live working log and frontend view
- Sales is inactive initially but reserved for expansion
- All agent code happens inside Performance tab live terminal area
- Each project has 3 AI agents managing tasks independently
- Titan does not assign tasks or handle coding; it oversees project health and alerts only

---

**Autonomy and Monitoring:**
- Each agent is separated and works independently to reduce overload
- Titan checks every project’s health every 20 minutes
- Green check means live coding is occurring (based on logs)
- Red X means the project is stalled and Titan will alert via Telegram
- Titan should monitor:
  - Live agent activity
  - Firebase storage (for overflow or space errors)
  - Inference usage or execution issues from AI agents

---

**Summary:**
Titan is a shell built on Replit for download. It is clean, modular, integration-ready, and designed for expansion post-download. Replit’s job is to make the interface and all non-AI logic and build connections that can be easily plugged into once the ZIP is imported into a VM.

It is not a public app. There is no login, no user accounts, and no database auth. It is for local use only.

The goal is to have this running autonomously within hours of download by plugging into Firebase, GPT, Telegram, and launching on a Google Cloud VM instance.

Replit must know: Titan is not meant to be built *on* Replit, only *by* Replit. Once exported, it lives somewhere else.
